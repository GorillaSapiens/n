If you're not respecting the ABI, you're one segfault away from chaos.

int foo(int in_sp, const int ref_in_sp, static int in_var_not_stack)

before call:
; push room for return+args onto ARGUMENT stack
lda #size
sta arg0
jsr _pushN
; populate return/arg area relative to caller fp
; stack-backed args occupy the frame in declaration order as seen by the callee,
; which means the caller writes the leftmost argument into the highest-addressed
; argument slot and the rightmost argument into the lowest-addressed slot.
; save caller frame pointer on HARDWARE stack
lda fp+1
pha
lda fp
pha
; jsr to target
jsr _foo
; restore caller frame pointer from HARDWARE stack
pla
sta fp
pla
sta fp+1
; copy return value out of caller's pushed area if needed
; pop return+arg area from ARGUMENT stack
lda #size
sta arg0
jsr _popN

function prologue:
; transfer current sp to fp
lda sp+1
sta fp+1
lda sp
sta fp
; allocate locals if any with pushN

function epilogue:
; pop locals if any with popN
rts

reset/startup:
; initialize both sp and fp to the initial argument-stack base
; because the argument stack grows upward, startup should seed them at __stack_start

A, P, X, and Y are caller-clobbered across ordinary function calls.
Callers must assume callees may freely overwrite them unless an API explicitly
documents stronger guarantees. The default calling sequence only preserves the
frame pointer so the callee can establish its stack frame and the caller can
recover it after the call.

NOTE: If you do have a special-case wrapper that needs to preserve both A and P,
and you want P exactly as it was, you must pull P last:
PHP      ; save status
PHA      ; save A
...
PLA      ; restore A (and flags get set from A)
PLP      ; restore original status flags, including N/Z
